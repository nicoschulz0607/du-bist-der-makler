# 🏠 brain.md — du-bist-der-makler.de

## 1. Vision

Eine Plattform für privaten Immobilienverkauf **ohne Makler** – aber mit allen Tools, die man braucht, um es professionell zu machen. Zwischen “ich stell’s auf ImmoScout” und “ich beauftrage einen Makler” gibt es eine echte Lücke. Diese Lücke füllen wir.

**Kern-Versprechen:** Du zahlst einmal. Du verkaufst selbst. Wir begleiten dich dabei.

PRD = Product Requirements Document — das Dokument das definiert was gebaut wird, für wen, und wie genau es funktionieren soll.

Es ist die Brücke zwischen brain.md (Ideen) und Code. Wenn du anfängst zu bauen und dein Kollege fragt "wie soll das genau funktionieren?" — das PRD ist die Antwort.

Konkret enthält es für jedes Feature:

Wer nutzt es und warum (User Story)
Was genau passiert (Akzeptanzkriterien)
Was es nicht tut (Out of Scope)
Welches Paket es freischaltet (Starter/Pro/Premium)
Ohne PRD baut man Features zweimal — einmal wie man es sich gedacht hat, einmal wie man es eigentlich wollte.

# DESIGN.md — du-bist-der-makler.de
 
> Drop this file into your project root. Reference it in your prompt to Cursor, Claude Code, or any AI agent:
> _"Refer to DESIGN.md for all UI decisions."_
>
> Based on the Airbnb design language (awesome-design-md/airbnb) — same structure, spacing, typography, and component logic. Brand colors replaced with du-bist-der-makler.de identity.
 
---
 
## 1. Design Philosophy
 
du-bist-der-makler.de feels like a **trusted advisor, not a portal**. The UI is warm, airy, and confident — like a well-designed real estate magazine that also happens to be a SaaS tool. Every screen communicates: _"You're in control, and you're not alone."_
 
**Core principles:**
- **White canvas, one accent.** #ffffff base. `--accent` (#1B6B45) is the only color that does work. Everything else is neutral.
- **Warm authority.** Inter at medium-to-bold weights only. No thin type. No cold tech aesthetic.
- **Generous breathing room.** Travel-magazine spacing. Users scroll slowly through a property sale — don't rush them.
- **Rounded warmth.** 12–20px border radius throughout. Pill buttons. Soft shadows. Nothing angular.
- **KI features are superpowers, not clutter.** AI tools get subtle badges, never flashy banners.
 
---
 
## 2. Color System
 
### Primary Palette
 
```
--white:          #ffffff   /* Canvas background — almost everything sits on this */
--accent:         #1B6B45   /* Rausch-equivalent: the ONE brand color for CTAs, active states, highlights */
--accent-hover:   #145538   /* Accent darkened ~15% for hover/press states */
--accent-light:   #E8F5EE   /* Accent at ~8% opacity — badges, tag backgrounds, subtle highlights */
--accent-mid:     #2D8A5A   /* Mid-tone for gradients or secondary accent use */
 
--text-primary:   #222222   /* Headlines, primary UI labels — near-black, warm */
--text-secondary: #717171   /* Captions, placeholders, metadata */
--text-tertiary:  #b0b0b0   /* Disabled states, very subtle hints */
 
--border:         #DDDDDD   /* Default border for inputs, cards, dividers */
--border-strong:  #B0B0B0   /* Focused inputs, selected states */
--surface:        #F7F7F7   /* Subtle page sections, table rows, sidebar backgrounds */
--surface-mid:    #EEEEEE   /* Stronger section separation */
```
 
### Semantic Colors
 
```
--success:        #1B6B45   /* Same as accent — success = brand */
--success-light:  #E8F5EE
--warning:        #C07000
--warning-light:  #FFF4E0
--error:          #C13515
--error-light:    #FDECEA
--info:           #1A3C6B
--info-light:     #E8EFF8
```
 
### Color Rules
 
- **Never use accent as a background for large areas.** Use `--accent-light` (#E8F5EE) for backgrounds; reserve `--accent` (#1B6B45) for buttons, active borders, and icons only.
- **Text on accent buttons is always #ffffff.**
- **Never mix accent with other saturated colors** in the same component. One color does the work.
- **Dark mode:** Not in scope for MVP. Design for light only.

## 3. Typography
 
### Font Stack
 
**Primary font:** Inter (Google Fonts, open source — closest open substitute for Airbnb Cereal VF)
 
```css

```

```
--text-lg:    22px / 32px   weight 600   /* Section subheadings */
--text-xl:    28px / 36px   weight 600   /* Section headings */
--text-2xl:   36px / 44px   weight 700   /* Page titles, hero subheadlines */

--text-4xl:   60px / 68px   weight 700   /* Large display (landing hero only) */
```
 
### Letter Spacing (Airbnb-style negative tracking on headings)
 
```
Display (48px+):    letter-spacing: -0.44px
Headings (28–44px): letter-spacing: -0.28px
Subheadings (22px): letter-spacing: -0.18px
Body:               letter-spacing: 0px (default)

```
 
### Weight Rules (Airbnb principle: always ≥ 500)
 
```
700 bold      — Hero headlines, pricing numbers, CTA button labels
600 semibold  — Section headings, card titles, nav active state
500 medium    — Body text, form labels, secondary nav, table headers
400 regular   — Captions, metadata, placeholder text only
```
 
**Never use weight 300 or lighter.** The brand voice is warm and confident — thin type feels cold and uncertain.

```
--space-1:   4px    /* Micro: icon-to-label gap, badge padding */
--space-2:   8px    /* Tight: input padding vertical, tag gap */
--space-3:  12px    /* Small: label-to-input gap */
--space-4:  16px    /* Base: card padding, list item gap */
--space-5:  20px    /* Medium: between form groups */
--space-6:  24px    /* Comfortable: section inner padding */
--space-8:  32px    /* Large: between card sections */
--space-10: 40px    /* XL: mobile section gap */
--space-12: 48px    /* 2XL: desktop section gap */
--space-16: 64px    /* 3XL: between major page sections */
--space-20: 80px    /* 4XL: hero vertical padding */
--space-24: 96px    /* 5XL: landing page section gap */
```
 
**Travel-magazine spacing rule:** Between major landing page sections, use `--space-24` (96px) desktop / `--space-16` (64px) mobile. Users are making a big decision — don't rush the page.

## 14. What NOT to Do
 
- ❌ Never use gradients as backgrounds (except the loading skeleton shimmer)
- ❌ Never use more than one accent color per screen
- ❌ Never use weight 300 or 400 for headings
- ❌ Never use border-radius < 4px (everything is rounded)
- ❌ Never use hard shadows (black, 100% opacity) — always use the three-layer rgba system
- ❌ Never put text directly on the accent color (#1B6B45) except white text
- ❌ Never use all-caps text except table headers and small badge labels
- ❌ Never use animation duration > 300ms for UI interactions
- ❌ Never use blue as a secondary brand color (ohne-makler.net owns blue in this market)
